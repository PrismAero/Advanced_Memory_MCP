# Contributing to AI Cognitive Suite

Thanks for your interest in contributing! Here's how you can help make this project better.

We accept changes through [the standard GitHub flow model](https://docs.github.com/en/get-started/using-github/github-flow).

## Development Workflow

Please follow our branching strategy:
- **develop** - Active development branch
- **staging** - Pre-production testing
- **main** - Production releases

All contributions should be made via pull requests to the `develop` branch.

## What We Welcome

We welcome:
- **Bug fixes** - Help us squash those pesky bugs
- **Performance improvements** - Making the servers faster and more efficient
- **Documentation improvements** - Better examples, clearer explanations
- **Test coverage** - Help us maintain code quality
- **Memory system enhancements** - Improvements to the branching knowledge graph
- **Reasoning system improvements** - Better structured thinking capabilities

## What We're Selective About

We're more selective about:
- **Major architectural changes** - Please discuss these in an issue first
- **New dependencies** - We prefer to keep the dependency footprint minimal
- **Breaking changes** - These require careful consideration and planning

## Code Quality

- All commits must be GPG signed
- Follow existing code style and patterns
- Include tests for new functionality
- Update documentation as needed

## Getting Started

1. Fork the repository
2. Create a feature branch from `develop`
3. Make your changes
4. Test thoroughly
5. Submit a pull request

## Questions?

Feel free to open an issue for discussion or contact kai@prismaquatics.io.

Thank you for helping make AI Cognitive Suite better!
